# NapCatQQ 安全增强说明

## 安全增强功能

本更新为 NapCatQQ WebUI 添加了全面的安全增强功能，以防止未经授权的访问和攻击。

### 新增安全功能

#### 1. 强化令牌生成
- **随机令牌**: 替换可预测的时间戳令牌为64字符的加密安全随机令牌
- **令牌复杂度验证**: 强制要求最小长度和复杂度
- **安全令牌生成器**: 提供工具生成强随机令牌

#### 2. IP访问控制
- **IP白名单**: 支持配置允许访问的IP地址列表
- **通配符支持**: 支持 `192.168.1.*` 形式的IP段匹配
- **自动阻断**: 不在白名单的IP将被拒绝访问

#### 3. 账户锁定机制
- **失败尝试计数**: 跟踪每个IP的登录失败次数
- **自动锁定**: 达到最大失败次数后临时锁定账户
- **可配置参数**: 支持配置最大失败次数和锁定时长

#### 4. 增强的安全头
- **XSS保护**: 设置 X-XSS-Protection 头
- **内容类型嗅探保护**: 设置 X-Content-Type-Options
- **点击劫持保护**: 设置 X-Frame-Options
- **内容安全策略**: 配置 CSP 头防止各种注入攻击
- **HSTS支持**: 在HTTPS模式下强制安全传输

#### 5. CORS安全增强
- **域名白名单**: 支持限制允许的跨域请求来源
- **更严格的策略**: 当配置域名白名单时，严格验证来源

#### 6. 配置安全化
- **默认localhost**: 将默认监听地址从 `0.0.0.0` 改为 `127.0.0.1`
- **安全配置项**: 新增专门的安全配置节
- **向后兼容**: 保持与现有配置的兼容性

### 使用方法

#### 基础配置
编辑 `webui.json` 配置文件：

```json
{
  "host": "127.0.0.1",
  "port": 6099,
  "token": "your-secure-64-char-random-token-here",
  "security": {
    "allowedIPs": [
      "127.0.0.1",
      "192.168.1.*",
      "10.0.0.0/24"
    ],
    "lockoutDuration": 30,
    "maxFailedAttempts": 5,
    "enableSecurityHeaders": true,
    "minTokenLength": 16,
    "forceHTTPS": false,
    "allowedOrigins": [
      "https://your-domain.com"
    ]
  }
}
```

#### Web界面配置
1. 登录 WebUI
2. 进入"配置" -> "安全配置"页面
3. 配置各项安全参数
4. 使用"生成安全令牌"工具创建强密码

### 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `allowedIPs` | 允许访问的IP地址列表，支持通配符 | `[]` (允许所有) |
| `lockoutDuration` | 账户锁定持续时间（分钟） | `30` |
| `maxFailedAttempts` | 触发锁定的最大失败次数 | `5` |
| `enableSecurityHeaders` | 是否启用安全头 | `true` |
| `minTokenLength` | 密码最小长度 | `16` |
| `forceHTTPS` | 是否强制HTTPS（需要SSL证书） | `false` |
| `allowedOrigins` | 允许的CORS来源域名 | `[]` (允许所有) |

### 安全建议

1. **立即更改默认密码**: 如果使用默认密码，请立即更改
2. **配置IP白名单**: 限制只有可信IP能够访问
3. **启用HTTPS**: 在生产环境中配置SSL证书并启用HTTPS
4. **定期更新密码**: 定期更换访问令牌
5. **监控访问日志**: 定期检查访问日志，识别异常访问

### 迁移指南

现有配置将自动兼容，无需手动迁移。但建议：

1. 检查安全状态：访问安全配置页面查看当前安全等级
2. 如显示使用默认密码，请立即更改
3. 根据需要配置IP白名单和其他安全选项

### API变更

新增安全配置API端点：
- `GET /api/Security/config` - 获取安全配置
- `POST /api/Security/config` - 更新安全配置  
- `POST /api/Security/generate-token` - 生成安全令牌
- `GET /api/Security/status` - 获取安全状态概览

### 技术实现

- **加密安全随机数**: 使用 Node.js `crypto.randomBytes()` 生成令牌
- **时间常数比较**: 防止时间攻击的安全密码比较
- **内存存储**: 失败尝试计数使用内存存储，重启后清零
- **中间件架构**: 安全检查通过Express中间件实现
- **类型安全**: 全面的TypeScript类型定义

这些安全增强功能显著提高了 NapCatQQ WebUI 的安全性，有效防止了通过扫描默认端口和使用默认密码的攻击方式。
